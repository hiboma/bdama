# ADR-0002: 技術スタックの選定

## ステータス

承認済み (2026-03-09)

## コンテキスト

「ビー玉ころころ」は幼児向けスマホブラウザゲームです。以下の技術要件があります。

- 2D 物理演算 (ビー玉の転がり、重力、衝突)
- タッチ入力 (指でパスを描画)
- アニメーション (敵の移動、障害物の回転、ボス演出)
- モバイルブラウザで 60fps 描画
- 外部アプリストア不要 (ブラウザのみで動作)

## 決定

### レンダリング: Canvas 2D API

DOM/CSS、PixiJS、Phaser 3 と比較し、Canvas 2D API を採用します。

| 候補 | バンドルサイズ | 学習コスト | パス描画 | 採否 |
|------|---------------|-----------|---------|------|
| Canvas 2D API | 0KB | 低 | ネイティブ対応 | 採用 |
| PixiJS | ~200KB | 中 | 追加実装必要 | 不採用 |
| Phaser 3 | ~1MB | 高 | 追加実装必要 | 不採用 |
| DOM/CSS | 0KB | 低 | 不向き | 不採用 |

Canvas 2D API はパス描画 (`lineTo`, `quadraticCurveTo`) をネイティブにサポートしており、指で描いたルートの描画に最適です。PixiJS や Phaser は本ゲームの規模に対してオーバースペックです。

### 物理演算: Matter.js

| 候補 | バンドルサイズ | 特徴 | 採否 |
|------|---------------|------|------|
| Matter.js | ~30KB | 軽量、ブラウザ特化、衝突検出 | 採用 |
| Planck.js | ~50KB | Box2D 移植、高精度 | 不採用 |
| 自前実装 | 0KB | 柔軟だが開発コスト高い | 不採用 |

Matter.js はビー玉の重力落下、パス上の転がり、障害物・敵との衝突検出をそのまま実現できます。Planck.js は精度が高い反面、本ゲームには過剰です。

### ビルドツール: Vite + TypeScript

| 候補 | HMR | 型安全 | 設定量 | 採否 |
|------|-----|--------|--------|------|
| Vite + TypeScript | 高速 | あり | 最小限 | 採用 |
| webpack + TypeScript | 低速 | あり | 多い | 不採用 |
| vanilla JS (ビルドなし) | - | なし | なし | 不採用 |

レベルデータ、物理パラメータ、ゲームステートの型定義により安全に開発できます。Vite の HMR でゲームパラメータ調整の開発効率も高いです。

### サウンド: Web Audio API

ブラウザネイティブの Web Audio API を使用します。外部ライブラリは不要です。

### レベルデータ: JSON

レベル構成 (障害物配置、敵配置、パラメータ) は JSON ファイルで管理します。レベルエディタを後から追加する余地を残します。

### ホスティング: GitHub Pages

静的ファイルのみで動作するため、GitHub Pages でホスティングします。CI/CD は GitHub Actions で構築します。

## 技術構成まとめ

```
フロントエンド: Vite + TypeScript
レンダリング:   Canvas 2D API
物理演算:       Matter.js
サウンド:       Web Audio API
レベルデータ:   JSON
ホスティング:   GitHub Pages
```

## アセット方針

モックで使用した CSS グラデーション描画を Canvas の描画コードに移植します。外部画像アセット (PNG/SVG) は使用せず、プログラムによる描画で統一します。これにより以下の利点があります。

- アセット読み込み時間がゼロです
- 解像度に依存しない描画ができます
- ファイルサイズを最小限に抑えられます

## リスク

- Canvas 2D API のパフォーマンスが不足する場合、PixiJS (WebGL) への移行を検討します
- Matter.js の物理演算がゲーム体験に合わない場合、パラメータ調整で対応し、それでも不十分なら自前実装に切り替えます
