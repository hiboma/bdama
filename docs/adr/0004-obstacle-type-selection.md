# ADR-0004: レベル制からの障害物タイプ選択制への変更

## ステータス

承認済み (2026-03-09)

## コンテキスト

元の設計（plan.md, ADR-0001）では Lv.1〜Lv.20 のレベル制で、レベルが上がるごとに新しい障害物や敵が登場する進行方式でした。幼児向けの遊びやすさを優先し、レベルの概念を取り下げてタイトル画面で障害物のタイプを選ぶ方式に変更します。

## 決定

### ゲームフロー

```
タイトル画面（障害物を選択）→ あそぶ → ゲームプレイ → クリア/失敗 → タイトルへ戻る
```

レベル進行は廃止し、1 ステージ構成にします。

### 障害物タイプ

タイトル画面で 4 種類の障害物タイプからプレイヤーが選択します。複数選択が可能です。

| タイプ | ラベル | 形状 | 色 | 物理特性 | アニメーション |
|--------|--------|------|----|----------|---------------|
| `rect` | しかく | 長方形 | 赤グラデーション | 静的な壁として配置します | ±20度でゆらゆら揺れます（回転軸: 重心/左端/右端からランダム） |
| `circle` | まる | 円形 | 青グラデーション | バンパーとして反発します | サイズが0.7倍〜1.3倍で脈動します |
| `triangle` | さんかく | 三角形 | 緑グラデーション | 静的な三角壁として配置します | ±40pxの上下移動をします |
| `cross` | くるくる | 回転する十字 | オレンジグラデーション | 毎フレーム回転し、ビー玉を弾きます | 0.02 rad/frame で回転します（方向はランダム） |

### 障害物サイズ

各障害物のサイズは以下のとおりです（画面幅に対する正規化値）。

| タイプ | サイズ計算 |
|--------|-----------|
| rect | 幅: `0.0675 + random * 0.045`、高さ: `0.015 + random * 0.0075` |
| circle | 半径: `0.0675 + random * 0.045` |
| triangle | 辺: `0.0675 + random * 0.045` |
| cross | サイズ: `0.0675 + random * 0.045` |

### 複数選択

- プレイヤーは 1 つ以上の障害物タイプを選択できます
- `Set<ObstacleType>` で選択状態を管理します
- 選択中のカードには金色のボーダーとチェックマークを表示します
- フィードバックメッセージ: 未選択時「いくつでも えらべるよ！」、選択時「Nこ えらんだ！」
- 複数選択した場合、障害物は選択したタイプに均等配分されます

### 障害物アニメーションの実装

各障害物は static body のまま、update ループで位置・角度・スケールを毎フレーム更新します。

- **rect**: `Matter.Body.setAngle` + `Matter.Body.setPosition` でピボット中心に回転させます。ピボットは重心・左端・右端からランダムに決定します。揺れの速度と位相も個体ごとにランダムです。
- **circle**: `Matter.Body.scale` で `circleRadius` を変化させます。物理的な当たり判定もサイズに追従します。
- **triangle**: `Matter.Body.setPosition` で Y 座標を sin 関数で変化させます。
- **cross**: Matter.js の複合ボディ（2 つの長方形を十字に組み合わせ）で構成します。`Matter.Body.rotate` で毎フレーム回転させます。回転方向は個体ごとにランダム（時計回り / 反時計回り）です。

すべてのアニメーションは playing / drawing / rolling の全状態で動作します。

### レインボーマーブル

各障害物タイプのステージに白玉の代わりとしてレインボーマーブルが 1 つ登場します。

| タイプ | 配置ルール |
|--------|-----------|
| rect | 障害物の上面に配置します |
| circle | バンパーの上方に配置します |
| triangle | 三角形の頂点付近に配置します |
| cross | 十字の腕先端に配置します |

レインボーマーブルは 7 色のコニックグラデーションで描画し、毎フレーム回転します。スパークルエフェクトが付きます。

### ゴール: ピアノバスケット

ゴールの見た目をピアノを模したかごに変更しました。

- 黒い台形のボディに白鍵 8 個と黒鍵のパターンを配置します
- 金色のトリムで縁取りします
- ゴール到達時に浮遊する音符（♪）アニメーションを表示します

### タイトル画面 UI

- 障害物カード 4 枚を横並びで表示します（カード幅 80px、高さ 110px、間隔 8px）
- 各カードにはプレビュー形状とひらがなラベルを表示します
- カードのプレビューはゲーム中と同じアニメーションをします
  - しかく: ±15度のゆらゆら揺れ
  - まる: 0.8倍〜1.2倍のスケール変化
  - さんかく: ±6pxの上下移動
  - くるくる: 回転
- カード下に「じゃまものを えらんでね」の説明テキストを表示します
- フィードバックメッセージは太字13pxの濃い色で表示します
- 1 つ以上選択すると「あそぶ」ボタンが有効になります

### ゲーム中の操作

- 棚の描画と端点のドラッグは playing / drawing / rolling すべての状態で可能です
- ビー玉が転がっている最中でも棚を追加・移動できます
- ゲーム画面の左下にリセットボタンともどるボタン（タイトルへ戻る）を配置します

### ビー玉の破壊

- 障害物との衝突によるビー玉の破壊は無効化しています（コードは残存）
- タップ4回によるビー玉の破壊は有効です

## 変更前との比較

| 項目 | 変更前 | 変更後 |
|------|--------|--------|
| 進行方式 | Lv.1〜Lv.20 のレベル制 | 1 ステージ、障害物タイプ選択制 |
| 障害物タイプ | レベルに応じて固定 | プレイヤーが複数選択 |
| 障害物の種類 | 2 種（rect, circle） | 4 種（rect, circle, triangle, cross） |
| 障害物の動き | 静的 | 各タイプ固有のアニメーション |
| 敵キャラ | ピエロ、リングマスター | 廃止 |
| ゴール | サーカスリング | ピアノバスケット |
| 白玉 | rect ステージのみ | 全タイプに登場（レインボーマーブル） |
| 棚のドラッグ | playing / drawing のみ | rolling を含む全プレイ状態 |
| ビー玉破壊 | 障害物衝突で破壊 | 衝突破壊は無効（タップ破壊は有効） |
| ナビゲーション | なし | もどるボタン（リセットの右隣） |

## 理由

- レベル制は幼児にとって「進められない」というストレスを生む可能性があります
- 障害物タイプの選択は「自分で選ぶ楽しさ」を提供します
- 複数選択により遊びの組み合わせが増え、リプレイ性が向上します
- 十字の回転障害物は視覚的に楽しく、ゲーム性を高めます

## 影響を受けるファイル

- `src/game/Game.ts` - ゲームフロー、障害物生成、状態管理
- `src/game/Renderer.ts` - タイトル画面 UI、障害物描画、レインボーマーブル、ピアノゴール
- `src/game/LevelManager.ts` - レベルデータの簡素化（1 レベルのみ）
- `plan.md` - ゲーム設計ドキュメント（要更新）
