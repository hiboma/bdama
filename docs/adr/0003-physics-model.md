# ADR-0003: 物理モデルの変更

## ステータス

承認済み (2026-03-09)

## コンテキスト

v1 のモックでは、ビー玉がプレイヤーの描いた縦方向のパスに沿って移動する設計でした。スタート（画面下部）とゴール（画面上部）の位置が固定されており、レベルデザインの自由度に制約がありました。

## 決定

物理モデルを以下のように変更します。

### 軸ごとの役割

| 軸 | 役割 | 制御方法 |
|----|------|----------|
| X軸 | 転がる | プレイヤーが描いた棚（シェルフ）の傾きで方向が決まります |
| Y軸 | 落下・はねる | 重力による自然落下と、トランポリンやバウンドで上方向に跳ねます |

### プレイヤー操作

プレイヤーは指で横方向に「棚」を描きます。ビー玉は棚の上を転がり、棚の端から落下して次の棚に着地します。棚の傾きによってビー玉の転がる方向（左右）が変わります。

### スタート/ゴール位置の可変化

v1 ではスタートが画面下部中央、ゴールが画面上部中央に固定されていました。v2 ではスタートとゴールの位置をレベルごとに自由に配置できます。

| レベル | スタート | ゴール |
|--------|----------|--------|
| Lv.1 | 左上 | 右下 |
| Lv.5 | 右上 | 左下 |
| Lv.10 | 上中央 | 右下 |
| Lv.15 | 左上 | 右下 |
| Lv.20 | 左上 | 下中央 |

### 新要素: トランポリン

ビー玉が上に跳ねるバウンドパッドを追加します。落下するだけでなく上方向への移動も可能にすることで、レベルデザインの幅が広がります。

### 新要素: 重力インジケータ

画面右端に下向きの矢印と「じゅうりょく」ラベルを表示し、幼児にも重力の方向が直感的に分かるようにします。

## v1 との比較

| 項目 | v1 | v2 |
|------|----|----|
| プレイヤー操作 | 縦方向のパスを描く | 横方向の棚を描く |
| ビー玉の動き | パス上を移動 | 棚の上を転がり、端から落下 |
| 物理モデル | パス追従 | 重力 + 衝突 + バウンド |
| スタート/ゴール | 固定（下→上） | レベルごとに可変 |
| 新ギミック | なし | トランポリン |

## 理由

- スタート/ゴールの位置が可変になることで、レベルデザインの自由度が向上します
- 重力による落下とバウンドは、幼児にとって直感的に理解しやすい物理です
- 横方向に棚を描く操作は、スマホの横スワイプと相性が良いです
- Matter.js の物理演算（重力、衝突、反発）をそのまま活用できます
