# ビー玉ころころ - ゲーム設計ドキュメント

## 概要

幼児向けスマホブラウザパズルゲームです。ビー玉が重力で落下する2D平面上で、プレイヤーが指で横方向の「棚」を描いてビー玉をゴールまで誘導します。

## テーマ

サーカスランド（ADR-0001 で決定）

## 物理モデル（ADR-0003 で決定）

- **X軸**: ビー玉が棚の傾きに沿って転がります
- **Y軸**: 重力で落下し、トランポリンで跳ねます
- スタート/ゴールの位置はレベルごとに可変です

## ゲームフロー

```
タイトル → レベル選択 → ゲームプレイ → クリア/失敗 → 次のレベル or リトライ
```

### ゲームプレイの流れ

1. 画面にスタート、ゴール、障害物が表示されます
2. プレイヤーが指で棚を描きます（複数本の棚を描画可能）
3. 「スタート」ボタンでビー玉が転がり始めます
4. ビー玉が棚の上を転がり、端から落下します
5. ゴールに到達するとクリアです
6. 画面外に落下するか敵に当たると失敗です

## ゲーム要素

### ビー玉

- ゴールドのガラス質感です
- 物理演算（重力、衝突、反発）で動きます
- 棚の上を転がり、端から落下します

### 棚（プレイヤーが描画）

- 指でスワイプして描きます
- 傾きによってビー玉の転がる方向が変わります
- 描画数の上限はレベルごとに設定します

### 障害物: ドラム

- 静的なものと移動するものがあります
- ビー玉が接触すると跳ね返ります

### トランポリン

- ビー玉が上方向に跳ねます
- 固定配置です

### 回転プラットフォーム

- 回転しながらビー玉を弾きます
- 中〜高難度レベルで登場します

### 敵キャラ: ピエロ

- 横方向にパトロールします
- ビー玉が接触すると失敗です

### ボス: リングマスター

- Lv.20 で登場します
- 横方向に大きくパトロールし、ゴール付近を守ります

## 難易度設計

| レベル帯 | 棚の数上限 | 障害物 | 敵 | ギミック |
|----------|-----------|--------|-----|---------|
| Lv.1-4 | 3本 | なし | なし | なし |
| Lv.5-9 | 4本 | 静的ドラム | なし | トランポリン |
| Lv.10-14 | 4本 | 静的+移動ドラム | ピエロ 1体 | トランポリン |
| Lv.15-19 | 5本 | 壁+移動ドラム | ピエロ 3体 | 回転プラットフォーム、トランポリン |
| Lv.20 | 5本 | 迷路壁+移動ドラム | ピエロ 2体 | ボス（リングマスター）、回転プラットフォーム |

## スコアリング

- 各レベルで星1〜3を獲得できます
- 星の基準: 棚の使用数が少ないほど高評価です
  - 星3: 最小描画数でクリア
  - 星2: 上限-1本以内でクリア
  - 星1: 上限ぴったりでクリア

## 技術スタック（ADR-0002 で決定）

- Vite + TypeScript
- Canvas 2D API（レンダリング）
- Matter.js（物理演算）
- Web Audio API（サウンド）
- JSON（レベルデータ）
- GitHub Pages（ホスティング）

## レベルデータ構造

```json
{
  "level": 1,
  "start": { "x": 0.1, "y": 0.05 },
  "goal": { "x": 0.9, "y": 0.9 },
  "maxShelves": 3,
  "obstacles": [],
  "enemies": [],
  "trampolines": [],
  "spinPlatforms": [],
  "stars": { "3": 1, "2": 2, "1": 3 }
}
```

座標は 0.0〜1.0 の正規化値で、画面サイズに依存しない設計です。
